# 대법원 나의 사건검색 연동 가이드

LawyGo에서 **사건번호**와 **당사자 이름**을 입력했을 때, 대법원/법원 쪽 기록을 가져와 연동하는 방법을 정리합니다.

---

## 1. 개요

- **나의 사건검색**: 대법원·법원에서 제공하는 서비스로, 사건번호·당사자명 등으로 소송 진행 상황을 조회할 수 있습니다.
- **연동 방식**은 크게 두 가지입니다.
  1. **공식 API(사법정보공유포털)** – 사건조회·사건진행내역 API 사용 (가입·신청 필요)
  2. **수동 조회 + 링크** – LawyGo에서 조회 페이지로 바로가기 링크 제공 (즉시 가능)

---

## 2. 공식 API 연동 (사법정보공유포털)

### 2.1 포털 안내

- **사이트**: [사법정보공유포털](https://openapi.scourt.go.kr/)
- **제공 내용**  
  - **데이터 API**: 판례, **사건정보** 등  
  - **조회 API**: **사건진행 내역** 등 조회  
  - 문건제출 관련 API (사실조회회신, 기일입찰조서 등)

### 2.2 이용 절차

1. **회원가입**  
   - [사법정보공유포털](https://openapi.scourt.go.kr/) 접속 → 회원가입 (개인/법인)
2. **API 신청**  
   - 포털 내 오픈 API·연계 API 안내에 따라 **사건조회 / 사건진행내역** 관련 API 신청
3. **승인 후**  
   - 인증키·엔드포인트·명세를 받으면, 아래 4장처럼 LawyGo에서 호출해 기록을 가져올 수 있습니다.

### 2.3 주의사항

- **개인정보**: API 이용 목적 범위 내 최소 수집, 목적 외 이용 금지
- **저작·재가공**: 대량 저장·재가공·상업적 이용은 이용약관 확인 필요
- **본인인증**: 일부 서비스는 공인인증서·금융인증서 등 추가 인증 요구 가능

정확한 신청 절차·파라미터(사건번호, 당사자명 등)는 **사법정보공유포털**의 최신 안내를 따르세요.

---

## 3. 수동 조회 + 링크 (즉시 사용)

API 없이도 **사건번호·당사자 이름**만 LawyGo에 있으면, “대법원 나의 사건검색” 페이지로 넘어가서 직접 조회할 수 있도록 할 수 있습니다.

### 3.1 나의 사건검색 접속 경로

- 대법원 포털:  
  [나의 사건검색](https://www.scourt.go.kr/portal/information/events/search/search.jsp)  
  또는 [ssgo.scourt.go.kr](https://ssgo.scourt.go.kr/ssgo/index.on?cortId=www)
- **검색 조건**: 관할법원, 사건년도, 사건구분, **사건번호**, **당사자명**, 보안코드(화면 입력) 등

### 3.2 LawyGo에서 할 수 있는 것

- 사건 상세/목록 화면에 **“법원에서 조회”** 같은 버튼을 두고,  
  위 나의 사건검색 URL을 새 탭으로 열어주기
- 필요하면 **사건번호·당사자명**을 클립보드에 복사해 두어, 사용자가 포털에 붙여넣기 쉽게 하기  
  (URL에 파라미터를 넣을 수 있으면, 추후 쿼리스트링으로 넘기는 것도 가능)

이렇게 하면 “사건번호와 당사자 이름을 치면” LawyGo에서 그 값을 가지고 **법원 조회 페이지로 연동**하는 형태를 만들 수 있습니다.

---

## 4. API 연동 시 LawyGo 구현 방향 (추후)

API 키·엔드포인트를 받은 뒤에는 아래와 같이 연동할 수 있습니다.

### 4.1 환경 변수

```env
# .env.local (API 승인 후 발급된 값으로 설정)
NEXT_PUBLIC_SCOURT_OPENAPI_BASE=https://openapi.scourt.go.kr
SCOURT_OPENAPI_KEY=발급받은_인증키
```

### 4.2 연동 흐름

1. **LawyGo 사건 등록/상세**  
   - 사용자가 **사건번호**, **당사자명**(원고/피고 등) 입력
2. **“법원 기록 가져오기” 버튼**  
   - 위 값으로 사법정보공유포털 **사건조회/사건진행내역 API** 호출
3. **응답 매핑**  
   - API 응답(진행단계, 기일, 접수일 등)을 LawyGo의 `CaseItem`·타임라인 등에 맞게 변환
4. **저장**  
   - 기존 사건 수정 또는 타임라인에 “법원 조회 결과”로 저장

### 4.3 코드 위치 제안

- **API 클라이언트**: `src/lib/scourtApi.ts`  
  - 인증키, base URL, 사건번호·당사자명 파라미터로 요청
- **서버 API 라우트**: `src/app/api/court-case/route.ts`  
  - 클라이언트에서 “가져오기” 요청 시, 서버에서만 `scourtApi` 호출 (키 노출 방지)
- **UI**  
  - 사건 상세 또는 사건 등록 폼에 “대법원 나의 사건검색에서 기록 가져오기” 버튼 → 위 API 호출 후 결과 반영

실제 **요청/응답 스키마**는 사법정보공유포털 API 명세를 받은 뒤 `scourtApi.ts`와 라우트에 맞춰 구현하면 됩니다.

---

## 5. 요약

| 방식 | 사건번호·당사자 입력 시 연동 방법 |
|------|----------------------------------|
| **즉시** | LawyGo에 사건번호·당사자 저장 → “법원에서 조회” 버튼으로 나의 사건검색 페이지 열기 (또는 복사 지원) |
| **API 연동** | 사법정보공유포털 가입·API 신청(사건조회/진행내역) → 승인 후 `scourtApi` + 서버 API + “기록 가져오기” 버튼으로 자동 반영 |

먼저 **수동 링크/복사**로 “사건번호·당사자 이름 치면 법원 조회와 연동해서 쓸 수 있게” 하고, API 승인 후 **자동 조회·기록 가져오기**를 추가하는 순서를 권장합니다.
