# 회계/수납 관리 · 다계좌 출입금 연동 기획안

## 1. 개요 및 목표

- **목표**: 회계/수납 관리 페이지에서 **여러 은행 계좌의 입출금 내역을 한 화면에서 동시에 확인·관리**할 수 있도록 한다.
- **사용자 시나리오**: 담당자가 “계좌 연결”로 사무소 계좌들을 등록한 뒤, 입금 내역을 한곳에서 보고 미수금 청구서와 매칭할 수 있다.
- **참고 UX**: 토스 앱처럼 계좌별로 구분해 보여 주고, 계좌명 클릭 시 해당 계좌 상세(거래내역)를 새 창/패널로 확인할 수 있게 한다.

---

## 2. API 연동 가능성 정리

### 2.1 토스 측 API

| 구분 | 내용 | 다계좌 출입금 연동 적합성 |
|------|------|---------------------------|
| **토스페이먼츠** | 결제·가상계좌·정산 API 제공. 가상계좌 입금 알림(웹훅), 정산 내역 조회 가능. | 토스로 들어온 결제/정산만 조회 가능. **일반 은행 계좌 입출금 통합 조회는 미제공.** |
| **오픈뱅킹(토스 문서)** | 토스 개발자 문서에서 오픈뱅킹 용어 사용. 잔액·**거래내역조회** 등 서비스 나열. | 실제 **거래내역 조회는 금융결제원 오픈뱅킹 API** 이용 전제. 토스가 “여러 은행 계좌 한번에” 제공하는 공개 API는 문서상 명확하지 않음. |

**정리**:  
- “토스 API만으로” 여러 은행 계좌의 출입금을 동시에 조회하는 공식 상품은 제한적이다.  
- **가능한 조합**:  
  - **(A) 토스페이먼츠**: 가상계좌·정산 API → 토스 경유 결제/입금만 연동.  
  - **(B) 금융결제원 오픈뱅킹 API**: 은행별 계좌 목록·잔액·**거래내역 조회** → 여러 은행 계좌를 실제로 “동시에” 조회·관리하려면 이쪽 연동이 필요하다.

### 2.2 권장 방향

- **기능 목표(여러 계좌 출입금 동시 확인·관리)** 를 만족하려면 **금융결제원 오픈뱅킹 API** 연동이 사실상 필요하다.  
- **토스**는 다음처럼 활용하는 구성을 추천한다.  
  - 토스페이먼츠: 가상계좌 입금·정산 데이터를 “계좌 하나”처럼 LawyGo에 표시.  
  - UI/UX: 토스 앱처럼 “계좌 카드 + 클릭 시 상세” 패턴을 차용.  
- 즉, **“토스 API 연동”** = 토스페이먼츠(결제/정산) + **오픈뱅킹(다계좌 거래내역)** 조합 기획안으로 정리하는 것이 현실적이다.

## 3. 제안 아키텍처

```
[ LawyGo 프론트 ]
  - 회계/수납 페이지: 계좌 목록 + 통합 입금 내역 + 기존 매칭 UI
  - 계좌 연결 버튼 → (오픈뱅킹 인증 또는 관리자 설정)
  - 계좌명 클릭 → 새 창/탭: 해당 계좌 거래내역 상세

[ LawyGo 백엔드 (Next API 등) ]
  - /api/finance/accounts      : 연동 계좌 목록 (DB + 캐시)
  - /api/finance/transactions : 통합 거래내역 (기간/계좌 필터, 페이지네이션)
  - /api/finance/accounts/[id]/transactions : 계좌별 거래내역 (새 창용)

[ 외부 연동 ]
  - 금융결제원 오픈뱅킹 API : 계좌 목록, 잔액, 거래내역 조회
  - (선택) 토스페이먼츠: 가상계좌 입금 알림·정산 내역 → 하나의 “계좌”처럼 취급
```

- **계좌 연결**:  
  - **오픈뱅킹**: 금융결제원 가입·승인 후, 사용자 인증(은행 로그인 등)으로 계좌 동의 받고 `account_id` 등을 저장.  
  - **토스페이먼츠**: 가상계좌/정산 정보를 “가상 계좌” 하나로 등록해 입금·정산 내역을 계좌별 목록에 포함.

## 4. UI/UX 설계

### 4.1 페이지 구조 (회계/수납 관리)

- 상단: 기존 유지 (제목, 미매칭 입금·미수금 건수, 통계 보기, 수납 등록).  
- **신규 블록**: “연동 계좌” (계좌 카드 영역).
  - **계좌 카드**
    - 표시: `은행명 · 마스킹된 계좌번호 · 잔액(선택)`  
    - 예: `국민은행 ****1234 · 1,234,000원`
  - **“+ 계좌 연결” 버튼**
    - 클릭 시: 오픈뱅킹 인증 유도 또는 관리자 전용 “계좌 등록” 플로우 (은행 선택 → 인증 → 계좌 선택 후 저장).
  - **계좌명(은행명·계좌번호) 클릭**
    - **새 창(또는 새 탭)** 오픈: ` /finance/accounts/[accountId] `  
    - 해당 계좌만의 **입출금 거래내역** 목록 (날짜, 적요, 입금/출금, 잔액 등) 표시.  
    - 토스 앱의 “계좌 탭 → 거래내역”과 유사한 느낌.

- 기존 “수납 매칭 인터페이스”는 유지.
  - **미확인 입금 내역** 출처를 “연동 계좌”로 확장:  
    - 각 row에 **어느 계좌에서 온 입금인지** 표시 (은행명·마스킹 계좌번호 또는 계좌 별칭).  
  - 미수금 청구서 ↔ 입금 내역 매칭 로직은 그대로 두고, 데이터만 “연동 계좌 거래내역 + 기존 수동 입력” 통합으로 확장.

### 4.3 계좌 상세(새 창) 화면

- **URL**: `/finance/accounts/[accountId]` (또는 쿼리 `?account=xxx`).
- **표시 내용**
  - 계좌 정보: 은행명, 마스킹 계좌번호, 잔액(오픈뱅킹에서 조회 시).
  - 기간 필터: 오늘, 1주, 1개월, 3개월, 직접 지정.
  - 테이블: 일자, 적요/메모, 입금, 출금, 잔액(선택).
- “이 내역을 수납 매칭으로 가져가기” 같은 액션을 두면, 메인 수납 화면의 “미확인 입금”에 해당 거래가 추가되는 식으로 연동 가능.

## 5. 데이터 모델 확장 제안

- **연동 계좌**
  - `id`, `bankCode`, `bankName`, `accountNumberMasked`, `accountHolder`, `displayName`, `source`(`openbanking` | `toss_virtual`), `refreshTokenOrApiKey`(암호화 저장), `lastSyncedAt`, `createdAt`.
- **거래 내역(입출금)**
  - 기존 `BankTransaction`에 `accountId`(연동 계좌 FK), `transactionType`(입금/출금) 등 필드 추가해 “어느 계좌 것인지”와 “출입금 구분”을 저장.

## 6. 구현 단계 제안

| 단계 | 내용 | 비고 |
|------|------|------|
| **1** | 기획·정책 확정 | 토스페이먼츠만 할지, 오픈뱅킹까지 할지, 관리자만 계좌 연결할지 등 |
| **2** | 연동 계좌 목록 UI | “연동 계좌” 블록, “+ 계좌 연결” 버튼, 계좌 카드(이름·마스킹번호·잔액). 클릭 시 새 창 라우트만 준비 |
| **3** | 새 창 계좌 상세 페이지 | `/finance/accounts/[id]` 목업. 목업 거래내역으로 테이블 UI 구현 |
| **4** | 미확인 입금에 `accountId` 반영 | 입금 내역 row에 “계좌: 국민****1234” 표시, 매칭 로직은 기존 유지 |
| **5** | 토스페이먼츠 연동(선택) | 가상계좌 입금 웹훅·정산 API → “토스 가상계좌” 하나를 연동 계좌로 등록 |
| **6** | 금융결제원 오픈뱅킹 연동 | 가입·승인 후 계좌 목록/거래내역 API 호출, 동기화 및 저장 |

---

## 7. 주의사항 및 리스크

- **금융결제원 오픈뱅킹**: 이용약관 체결·승인 필요. 개발/테스트 시 테스트 기관·테스트 API 여부 확인.
- **보안**: 계좌 연동용 토큰·키는 암호화 저장, API는 서버에서만 호출(클라이언트에 노출 금지).
- **토스 정책**: 토스페이먼츠는 결제/정산 위주이므로 “여러 일반 은행 계좌 출입금 통합”은 오픈뱅킹에 의존한다는 점을 이해하고 기획하는 것이 좋음.

## 8. 요약

- **가능 여부**:  
  - “여러 계좌의 출입금을 동시에 확인·관리”하는 기능은 **금융결제원 오픈뱅킹 API**를 연동하면 구현 가능하다.  
  - **토스 API**만으로는 토스 경유 결제/정산만 다루는 수준이므로, “다계좌 통합”은 오픈뱅킹과 조합하는 기획이 적합하다.
- **UI**:  
  - **계좌 연결**: 상단 “연동 계좌” 영역 + “+ 계좌 연결” 버튼.  
  - **토스 스타일**: 계좌 카드(은행명·마스킹 계좌번호)를 두고, **계좌명(카드) 클릭 시 새 창**에서 해당 계좌 거래내역만 보여 주는 구조를 추천한다.
- **다음 단계**:  
  - 2~4단계까지는 **API 연동 없이** UI·라우트·목업 데이터만으로 진행 가능하다.  
  - 5~6단계에서 토스페이먼츠·오픈뱅킹 API를 순차 도입하면 된다.

이 기획안을 기준으로 단계별로 개발 범위를 나누어 적용하시면 된다.
